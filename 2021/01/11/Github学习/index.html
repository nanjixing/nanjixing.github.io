4.2 分支操作

​ 进行多个并行作业时使用

​ 1.分支的创建

​ master分支使主干分支，一个版本发布有一个。如果有多个版本发布应该有多个。

​ 2.分支的合并

​ git merge

​ 分支合并流程首先进入要合并的分支的主分支

4.3 更改提交的操作，回溯和冲突合并

​ git reset 回溯历史版本

​ 1.git reset --hard 时间点的hash值 要先进入到要合并的主分支比如master分支

​ git log查看的是以当前状态为终点的历史日志

​ git reflog查看的是当前仓库执行过的操作日志

​ 2.合并时的冲突

​ 一般要删除其中之一，我们进行更改之后再次add commit即可

​ 3.修改提交信息

​ git commit --amend实际操作时我弹出了一个⚠选项选E继续进行即可进入文本编辑同前

​ Tips：进入仓库时现金仓库所在目录再进仓库

​ 4.压缩提交之前的错误历史

​ 1.git rebase -i HEAD~2选定当前分支中包含HEAD在内的两个最新历史记录为对象

​ 2.在弹出的编辑器中将要合并的文件的pick字段改为fixup

4.4 推送至远程仓库

​ 首先在github上创建仓库是不要选择readme文件，只创建一个空项目

​ git remote add origin ssh链接自动将远程仓库的名称设置为origin(标识符)

​ git push 推送至远程仓库

​ git push -u origin master master是我们当前所在的分支，如果在其他分支可以替换，-u参数可以将origin仓库的master（可以替换）分支作为本地仓库master(可以替换)当前分支的upstream,这样我们本地仓库获取是可以直接获取，见下。

4.5 从远程仓库获取

​ 1.git clone ssh链接要另外换一个文件夹用于模拟不同用户

​ 默认到master分支可以使用git branch -a命令进行查看当前分支本地仓库和远程仓库信息

​ 2.git checkout -b 本地分支名 origin/远程仓库分支名会创建一个和远程仓库分支内容完全一样的分支并进入该分支

​ 3.再次推送直接git push即可

​ 4.git pull origin 分支名将本地分支更新 到和远程仓库的分支的最新状态
